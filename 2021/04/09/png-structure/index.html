png structure

png图片-结构

1. 概述

Portable Network Graphics (PNG) is a raster graphics file format that supports lossless data compression.

  • jpg：有损压缩的图片格式，
    • 优点： 尺寸相对较小；符合人的视觉感知；保留图片原有光度同时擦除部分的色度信息
    • 缺点：不适用客户端
  • png：无损压缩的图片格式
    • 优点：客户端应用广； 支持多种类型：png8/png24/png32
    • 缺点：图片较大，更消耗带宽和存储

2. 结构

结构概述

png由header(文件头)和chunk(数据块)两部分组成

  1. header：文件头是固定取前八个字符，主要作用是存储文件格式数据，也称为魔数，固定内容为：[0x89, 0x50, 0x4E, 0x47, 0x0D, 0x0A, 0x1A, 0x0A]
  2. 数据块：数据块由不同类型组成，大类分为关键数据块和辅助数据块；关键数据块决定了图形的格式、长度、长宽、压缩、内存等
    • 关键数据块：IHDR PLTE IDAT IEND
数据块结构

数据块格式如下：

描述 长度
数据块长度 4字节
数据块类型 4字节
数据块内容 x字节
crc冗余校验码 4字节
IHDR：文件头数据块

用于存放 图片信息，长度固定为13个字节

描述 长度
图片宽度 4字节
图片高度 4字节
图形深度 1字节
颜色类型 1字节
压缩方法 1字节
过滤方法 1字节
扫描方式 1字节

图像深度：每个像素点每个通道的位数 1、2、4、8 和 16

颜色类型：每个像素点的通道数量 0、2、3、4 和 6

​ 0：灰度通道； 2：rgb真彩色通道； 3：索引通道； 4：灰度+alpha通道；6：4argb通道

PLTE：索引颜色

索引值

plte也称为”调色板“，当图片类型为索引图片时起作用

通常使用索引颜色的情况下，图像深度的值即为 8，因而调色板里存放的颜色就只有 256 种颜色

长度 = 2^(图像深度) * 3

IDAT:数据块内容

该类型数据库在png图片中存在多个且连续，显示图片时必须读取连续的IDAT内容

IEND:数据块结束内容

读取到该类型的数据块时表示读取数据完成

3. 无损压缩策略

图片的压缩跟颜色种类和图片类型、颜色类型、支持Alpha相关性较大

  1. 当图片颜色种类小于256种的情况，改用png8类型
  2. 当图片没有alpha颜色通道的情况，改用png24类型
  3. 当图片rgb三色通道值相同的情况，改用灰度格式(png8)
  4. 当图片所有alpha通道值等于1的情况， 删除透明通道

4. 有损压缩策略

  1. 当图片包含alpha通道的情况，擦除透明图片rgb通道色值
  2. pngquant核心算法：矢量量化计算，可以理解为：将颜色分组，合并同组色值相近像素，减少颜色种类
