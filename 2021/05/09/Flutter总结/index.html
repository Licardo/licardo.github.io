      Flutter总结

      Post on 2021-05-09

      Dart语言的优势

      1. JIT与AOT
      • JIT即just in time，开发测试效率高，运行速度和执行性能会受编译的影响
      • AOT即ahead of time，生成直接运行的二进制代码，运行速度快，执行性能好，开发测试效率低
      2. 内存分配和垃圾回收

      Dart VM的内存分配策略比较简单，只需要在堆上移动指针，内存增长是线性的，不需要查找可用内存的过程

      并发Isolate不共享内存的机制让Dart实现无锁快速分配内存

      Dart的垃圾回收采用多生代算法。新生代采用“半空间”机制即标记复制算法，触发垃圾回收时，Dart将当前半空间中标记的活跃对象拷贝到备用空间，然后释放当前空间的所有内存。回收过程只需要操作少量活跃对象，这种回收机制很适合Flutter框架中大量Widget销毁重建的场景

      3. 单线程模型

      抢占式需要锁来保护共享资源，但锁本身可能会带来性能损耗，甚至可能出现死锁等严重问题。

      Dart中没有多线程，只有Isolate，Isolates之间不共享内存，通过Event Loop在 Event Queue上传递消息通信

      我认为Flutter只是UI系统，不会涉及过多复杂的多线程场景

      4. 无需单独的声明式布局语言

      Dart的声明式编程布局易于阅读和可视化，不需要类似JSX或XML的声明式布局语言

      开发过程也不需要可视化界面编辑器，热重载可以很快在手机上看到运行效果

      5. Dart的未来

      随着Flutter和Fuchsia的推动，Dart SDK的迭代速度更快，Dart的生态正变得更加成熟

      Flutter优势

      1. 跨平台 - 提高代码复用，抹平了平台差异性带来的开发效率问题
      2. 高保真 - 底层渲染到上层开发语言的完整解决方案
      3. 高性能 - 代码执行效率和渲染性能媲美原生app

      Widget

      Flutter将Widget设计成不可变的，当视图渲染配置信息发生变化时，Flutter会选择重建Widget树的方式进行数据更新，以数据驱动UI构建的方式简单高效。

      缺点是涉及大量Widget对象的销毁和重建，对内存分配和垃圾回收会造成压力。不过Widget本身并不涉及实际渲染位图，所以只是轻量级的数据结构，重建的成本很低

      另外，由于Widget的不可变性，可以以较低成本进行渲染节点复用，因此在真实的渲染树中可能存在不同的Widget对应一个渲染节点(Element)的情况，这无疑有降低了重建UI的成本

      Element

      因为Widget具有不可变性，但Element却是可变的。Element树这一层将Widget树的变化做了抽象，可以只将真正需要修改的部分同步到真实的RenderObject树中，提高渲染效率，而不是销毁整个渲染视图树重建

      RenderObject

      Flutter采用深度优先遍历机制渲染对象树，确定树中各个对象的位置和尺寸，并把他们绘制到不同的图层上，绘制完毕后，合成和渲染的工作交由Skia完成
